# inha_db (Todolist 서비스 데이터베이스 설계)

인하대학교 4학년 1학기 Database(데이터베이스) 수업의 **DB 설계 및 구현 과제**를 위한 레포지토리입니다.

## 프로젝트 개요
* **서비스명**: Todolist (소셜 일정 관리 서비스)
* **목표**: 단순한 개인의 일정 관리를 넘어, 친구들과 일정을 공유하고 서로의 캘린더를 함께 관리할 수 있는 상호작용 기반의 일정 관리 서비스 설계 및 데이터베이스 구축

## 포함된 산출물 문서
* [`서비스정의서.md`](./서비스정의서.md): 서비스 구성 요소(사용자, 환경, 시스템, 기능) 상세 정의
* [`요구사항명세서.md`](./요구사항명세서.md): 요구사항 분석 및 개념적 설계 (개체, 속성, 관계 추출 및 E-R 다이어그램 구조화)
